# WB-Parser (Асинхронный парсер Wildberries CDN)

Асинхронный инструмент для быстрого получения данных о товарах с маркетплейса Wildberries по их артикулам (SKU). 
В отличие от стандартного браузерного парсинга HTML, данный скрипт напрямую опрашивает CDN-корзины WB (`basket-01` ... `basket-35`) для сверхбыстрого нахождения JSON-карточек товаров.

## Основные особенности
- **Обход блокировок**: Использует библиотеку `curl_cffi` для спуфинга TLS-отпечатков (имитация Chrome 120), что позволяет эффективно обходить базовые защиты и WAF (Web Application Firewall).
- **Молниеносный поиск**: Одновременный асинхронный опрос множества корзин. Не ждем ответа от всех серверов, парсинг идет с первого же успешного узла (`asyncio.as_completed`).
- **Моделирование данных**: Проверка, структурирование и типизация извлеченных данных с помощью `pydantic`.
- **Настройка поведения**: Вынесенные лимиты и управление сессиями (например, настройки `USER_AGENT`) через файл конфигурации `config.py`.

## Стек технологий
- Python 3.10+ (корректная поддержка Event Loop в Windows)
- `curl_cffi` (запросы с поддельными TLS-отпечатками)
- `asyncio` (асинхронная архитектура)
- `pydantic` (строгая типизация)

## Структура проекта
- `main.py` — Точка входа в программу. Здесь содержится список артикулов для парсинга и запуск главного асинхронного цикла.
- `client.py` — Логика HTTP-клиента `WBClient`. Формирует необходимые ссылки на основе артикула (vol, part), опрашивает "корзины" и возвращает Pydantic-модель.
- `models.py` — Описание схемы данных товара (класс `WBItem`).
- `config.py` — Глобальные параметры (User-Agent, таймауты и лимиты запросов).
- `requirements.txt` — Список зависимостей проекта.

## Установка

1. Перейдите в корневую директорию проекта, если еще не сделали этого.

2. Создайте и активируйте виртуальное окружение:
```bash
# Для Windows
python -m venv venv
.\venv\Scripts\activate

# Для Linux/MacOS
python3 -m venv venv
source venv/bin/activate
```

3. Установите необходимые зависимости:
```bash
pip install -r requirements.txt
```

## Использование

В текущей версии скрипта список целевых товаров (артикулов) задается напрямую в файле `main.py` в списке `articles_to_parse`. Добавьте или измените там нужные SKU-номера.

Для запуска парсера выполните команду:
```bash
python main.py
```
