# Парсер Wildberries (Асинхронный)

Асинхронный парсер данных с маркетплейса Wildberries.
Проект создан для портфолио, чтобы показать владение архитектурой Python, шаблонами проектирования и чистым кодом.

## Основные возможности
- **Асинхронный HTTP-клиент**: Использует `aiohttp` для неблокирующих запросов.
- **Обработка ошибок**: Паттерн Retry (через `tenacity`) для обхода лимитов (HTTP 429) и сбоев сети.
- **Валидация данных**: Строгая типизация моделей через `pydantic`.
- **Сохранение данных**: Параллельный экспорт в CSV-файлы и базу данных SQLite.
- **Пагинация**: Автоматический переход по страницам с задержками для защиты от базовых блокировок ботов.
- **CLI Интерфейс**: Удобный запуск с аргументами командной строки через `argparse`.

## Стек технологий
- Python 3.10+
- `aiohttp` - Асинхронные HTTP-запросы
- `pydantic` - Валидация данных
- `loguru` - Логирование
- `tenacity` - Механизм повторных попыток
- `pytest` - Юнит-тестирование
- `sqlite3` - Локальное хранение данных

## Установка

1. Создайте виртуальное окружение:
```bash
python -m venv venv
```

2. Активируйте окружение:
- Windows: `.\venv\Scripts\activate`
- Linux/MacOS: `source venv/bin/activate`

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

## Использование

Запустите парсер через командную строку, используя `main.py`.

```bash
python main.py "ноутбуки" -p 3
```

### Доступные аргументы:
- `query`: Поисковой запрос (например, "кроссовки", "ноутбуки").
- `-p`, `--pages`: Количество страниц для парсинга (по умолчанию: 1).
- `--no-csv`: Отключить сохранение данных в CSV файл.
- `--no-db`: Отключить сохранение данных в базу SQLite.

## Архитектура
- **Клиент (`src/client.py`)**: Управляет сессией и логикой HTTP-запросов.
- **Парсер (`src/parser.py`)**: Управляет логикой API запросов и преобразует JSON в модели Pydantic.
- **Модели (`src/models.py`)**: Описывает схемы данных (карточка товара, ответ поиска).
- **Хранилище (`src/db.py`, `src/exporters.py`)**: Отвечает за сохранение проверенных данных.

## Примечание о защите API
Wildberries активно борется с автоматическими запросами. При частом использовании вы можете столкнуться с ошибкой HTTP 429 (Слишком много запросов) или требованием пройти Капчу.
